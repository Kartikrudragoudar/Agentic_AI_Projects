AGENT 1 — FETCHER (agents/fetcher/app.py):

Purpose:
  Fetch live prices, % change, volume for all tickers in watchlist.json
  Fetch top 3 news headlines per ticker
  Save combined result to data/fetched.json

Libraries (all free):
  yfinance          → Indian + US stock prices, % change, volume, 52w high/low
  requests          → CoinGecko API for crypto prices
  feedparser        → RSS feeds for Indian market news

Data sources:
  yfinance library:
    ticker = yf.Ticker("RELIANCE.NS")
    info = ticker.fast_info
    Extracts: last_price, previous_close, pct_change, volume,
              day_high, day_low, week_52_high, week_52_low
    Handles both .NS (NSE) and plain US tickers

  CoinGecko API (free, no key needed, 30 calls/min):
    GET https://api.coingecko.com/api/v3/simple/price
    ?ids=bitcoin,ethereum,solana,matic-network
    &vs_currencies=inr,usd
    &include_24hr_change=true
    &include_market_cap=true
    Extracts: price_inr, price_usd, pct_change_24h, market_cap

  News RSS feeds (free, no key, unlimited):
    Moneycontrol:  https://www.moneycontrol.com/rss/latestnews.xml
    Economic Times: https://economictimes.indiatimes.com/markets/rss.cms
    Livemint:      https://www.livemint.com/rss/markets
    CoinDesk RSS:  https://www.coindesk.com/arc/outboundfeeds/rss/
    Parse with feedparser, filter by ticker/company name mention
    Return top 3 headlines per ticker

Logging:
  logger = logging.getLogger("fetcher")
  logging.basicConfig(
      level=logging.INFO,
      format="%(asctime)s [%(levelname)s] %(name)s: %(message)s"

Error handling:
  Wrap each ticker fetch in try/except
  If one ticker fails: log warning, continue others
  If no data at all: write empty fetched.json, log error — do NOT crash

AGENT 2 — SENTIMENT (agents/sentiment/app.py):

Purpose:
  Read data/fetched.json
  For each ticker, send price data + news headlines to Groq LLM
  Get back: sentiment label + one-line reason + confidence score
  Save to data/sentiment.json

LLM Integration (Groq — free, 14,400 req/day):
  from groq import Groq
  client = Groq(api_key=os.environ["GROQ_API_KEY"])

  Model: llama3-70b-8192
  Temperature: 0.2 (low = consistent, focused output)
  Max tokens: 200 per ticker

System prompt:
  "You are a financial market analyst. Given stock/crypto price data
  and recent news headlines, respond ONLY with valid JSON in this format:
  {
    'sentiment': 'Bullish' or 'Bearish' or 'Neutral',
    'confidence': 0-100,
    'reason': 'One sentence explaining the sentiment'
  }
  Base your analysis on price movement + news tone. Be concise."

User prompt per ticker:
  "Ticker: {name}
   Price change: {pct_change}%
   News headlines:
   - {headline_1}
   - {headline_2}
   - {headline_3}
   Analyse market sentiment."

Retry logic:
  MAX_RETRIES = 3
  On rate limit: wait exponentially (2s, 4s, 8s) then retry
  On failure after retries: sentiment = Neutral, confidence = 0,
  reason = "Sentiment analysis unavailable"
  Never crashes — always writes output even if LLM fails

AGENT 3 — SCORER (agents/scorer/app.py):

Purpose:
  Read data/fetched.json + data/sentiment.json + data/watchlist.json
  Score every ticker 0-100 based on multiple signals
  Generate alerts for unusual activity
  Save to data/scored.json

No LLM — pure Python logic, deterministic, free.

Scoring formula (total 100 points):

  Sentiment score (40 points):
    Bullish  + confidence → map to 25-40 pts
    Neutral  + confidence → map to 10-24 pts
    Bearish  + confidence → map to 0-9 pts

  Price momentum score (30 points):
    pct_change >= +3%  → 30 pts
    pct_change >= +1%  → 22 pts
    pct_change >= 0%   → 15 pts
    pct_change >= -1%  → 8 pts
    pct_change < -1%   → 0 pts

  Volume signal score (15 points):
    volume > 2.5x average → 15 pts (unusual — big move likely)
    volume > 1.5x average → 10 pts
    volume normal         → 5 pts

  Sector preference score (15 points):
    Ticker sector in preferred_sectors list → 15 pts
    Sector not in preferences              → 5 pts

Alert generation:
  Generate alert if any of:
    abs(pct_change) >= alert_on_price_change_pct (default 3%)
    volume > volume_spike_multiplier × avg_volume (default 2.5×)
    price crossed 52-week high
    price crossed 52-week low

Output (data/scored.json):
  {
    "scored_at": "...",
    "overall_market_mood": "Cautious",
    "scores": {
      "RELIANCE.NS": {
        "score": 82,
        "rank": 1,
        "highlight": true,
        "alerts": []
      },
      "HDFCBANK.NS": {
        "score": 45,
        "rank": 6,
        "highlight": false,
        "alerts": ["Volume 3.1x above average"]
      }
    }
  }

DOCKER COMPOSE (docker-compose.yml):

version: "3.9"

services:

  fetcher:
    build: ./agents/fetcher
    container_name: agent_fetcher
    env_file: .env
    volumes:
      - ./data:/data
    restart: "no"

  sentiment:
    build: ./agents/sentiment
    container_name: agent_sentiment
    env_file: .env
    depends_on:
      fetcher:
        condition: service_completed_successfully
    volumes:
      - ./data:/data
    deploy:
      resources:
        limits:
          memory: 512M
    restart: "no"

  scorer:
    build: ./agents/scorer
    container_name: agent_scorer
    depends_on:
      sentiment:
        condition: service_completed_successfully
    volumes:
      - ./data:/data
    restart: "no"

  formatter:
    build: ./agents/formatter
    container_name: agent_formatter
    env_file: .env
    depends_on:
      scorer:
        condition: service_completed_successfully
    volumes:
      - ./data:/data
      - ./output:/output
    restart: "no"

  backend:
    build: ./backend
    container_name: stock_backend
    env_file: .env
    ports:
      - "8000:8000"
    volumes:
      - ./data:/data
      - ./output:/output
    restart: unless-stopped

CRITICAL:
  depends_on condition: service_completed_successfully
  Waits for each container to EXIT before starting next.
  Without this, next agent starts before previous writes its file.
  restart: "no" for all agents — they run once and exit.
  restart: unless-stopped for backend — it runs continuously.

KEY DESIGN DECISIONS:

1. Groq instead of OpenAI — completely free, llama3-70b is excellent
   for sentiment classification tasks on financial headlines

2. RSS feeds instead of NewsAPI — unlimited and free vs 100/day limit
   Moneycontrol + Economic Times cover Indian markets comprehensively

3. service_completed_successfully in depends_on — waits for each
   container to EXIT before starting next, prevents race conditions

4. Backend as a separate always-on container — agents run and exit,
   backend serves the data to React continuously

5. Horizontal scroll brief section — inspired by floema, adds the
   premium immersive feel without complex WebGL for that section

6. Three.js via @react-three/fiber — cleaner React integration than
   raw Three.js imperative code, easier to maintain

7. Lenis duration 1.4 — slightly slower than default gives that
   luxury smooth scroll feeling like floema

8. Custom cursor — signature detail of premium 3D websites,
   immediately sets your site apart from generic dashboards
